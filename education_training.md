# 教育訓練：如何在 n8n 中串接 MCP

## 什麼是 n8n？

n8n 是一款開源的工作流程自動化工具，它讓你可以透過視覺化介面快速建立自動化流程，串接各種第三方服務和 API。無需撰寫大量程式碼，便可將多個應用和服務自動化整合。

## 什麼是 MCP？

MCP（My Cloud Platform）是一個雲端管理平台，提供多種API介面，方便用戶進行系統監控、資源管理及業務整合。

## n8n 串接 MCP 的基本步驟

### 1. 確認 MCP API 權限與憑證

請先取得 MCP 的 API Key 或相關存取憑證，這些通常由你的平台管理後台取得。

### 2. 建立 HTTP 請求節點 (HTTP Request Node)

1. 在 n8n 編輯頁面中，新增一個「HTTP 請求」節點。
2. 配置以下參數：
   - **HTTP 方法**：依 MCP API 文件選擇(GET, POST, PUT 等)
   - **URL**：MCP API 的端點網址。
   - **認證方式**：如需身份驗證，選擇相應的認證（API Key, Basic Auth 等），並填入你的憑證。
   - **Request Payload**：依需求填入 JSON 或其他格式的資料。

### 3. 連接節點並測試

將 HTTP 請求節點與前後流程節點連接，執行並檢視回應結果，確認成功串接。

### 4. 錯誤處理與優化

使用條件判斷節點或錯誤捕捉節點，提升流程可靠性。

## 範例流程

假設我們想從 MCP 取得某資源列表：

1. 新增一個 HTTP 請求節點，設定方法為 GET。
2. URL 設為 `https://api.mcpcloud.com/v1/resources`（依實際 API 修改）。
3. 在 Header 中加入 `Authorization: Bearer YOUR_API_KEY`。
4. 將輸出連接到一個 Debug 節點，查看輸出結果。

## 小結

透過 n8n 的 HTTP 請求節點，我們可以輕鬆將 MCP 平台的服務串接到自動化流程中。掌握 API 認證與資料格式是成功串接的關鍵。

若有需要，還可進一步利用 n8n 的豐富節點組合，實現更複雜的業務自動化。祝你學習順利！
